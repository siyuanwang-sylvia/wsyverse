## 第二视觉体系：宇宙海洋（The Archipelago）

**用于**：The Archipelago（原 Passion Lab）

**灵感**：深空星图 / 宇宙海洋 / 漂浮群岛 / 极光 / 星轨 / 文明遗迹 / 一个人的宇宙档案馆

**气质**：孤独但不冰冷 / 理性但浪漫 / 科幻但有人性 / 档案馆感 / 深海+宇宙 / 微弱呼吸感 / "文明遗迹"感

### 核心概念
- 整个模块 = 一片宇宙海洋
- 每个爱好 = 一座漂浮岛屿（不是分类，是地貌）
- 文件内容驱动岛屿演化（生长感 / 探索感 / 收藏感）
- 宇宙语言体系（Islands / Echoes / Fragments / Archives / Field Notes / Signals / Relics）

### 色彩
- 主背景：`#030510`（宇宙深渊黑）
- 星空：白色低透明度粒子
- 星云：极低透明度的紫/蓝/青色大光斑
- 极光：青/紫/翠绿渐变光带
- 岛屿光晕：基于 hash 的 HSL 色系
- 正文色：`#b8c8d8`
- 弱文本：`#5a7088`
- 强调：`#d0dae8`
- 边框：`white/[0.06~0.12]` 微弱白色

### 演化系统
- 0 文件 → UNCHARTED（未发现，雾中）
- 1-5 → DISCOVERED（初现）
- 6-15 → SETTLING（定居，结构出现）
- 16-30 → THRIVING（繁盛，生态完整）
- 31+ → CIVILIZATION（文明，自成一界）

### 雾系统
- < 3 天未更新：无雾
- 3-14 天：轻雾
- 14-30 天：中雾
- 30+ 天：重雾覆盖

## 首页交互逻辑
首页是"四道世界之门"，不是菜单。
- hover：门内出现动态微光 / 模糊背景浮现 / 空间流动感
- click：镜头推进 / 缓慢进入 / 沉浸式推进 / 不突然跳转

## 输出规范（每次开发必须遵守）

每次开发，先输出：
1. 本次模块目标
2. 页面空间设计思路
3. 动线逻辑
4. 文件结构
5. 后期扩展方案

**然后**再输出完整代码。不要直接开始写代码。

---

## 代码规范
- 始终输出完整 `page.js`，不给局部代码
- 默认使用 Tailwind CSS + framer-motion
- 遵循 Next.js App Router 结构（`"use client"` 按需添加）
- 路由跳转用 `useRouter`，不用 `window.location.href`
- 代码必须真实可运行
- 必须预留数据文件读取结构
